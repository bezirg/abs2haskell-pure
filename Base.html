The values of our language

type Ref = Int

We only have one type for our values: the Reference Both objects and futures use this type Ref. Since in this ABS-subset we are dynamically-typed, it is undefined what happens if an object is expected in place of a future, and vice-versa.

type ObjRef = Ref

Objects have type Ref

type FutRef = Ref

Futures have type Ref

type Counter = Ref

An ever-increasing counter to pick new unique references when creating new objects (via new) and futures (via async) (written as type-synonym just for clarity)

The datastructures used at ABS-runtime

data Heap

the simulated Heap datastructure where all program's objects live in. We assume no (automatic) Garbage-Collection of the Heap for the moment. In reality, no GC will be performed from the underlying Haskell-GC, because we do not delete objects

type Objects = Map ObjRef Attrs

The objects of the heap is a table _from_ an object's Reference _to_ another table of the object's attributes: Attrs

type Attrs = Map String Ref

The attributes is a table _from_ the attribute name (String) _to_ its value (Ref)

type Futures = Map FutRef (Maybe Ref)

The futures of the heap is a table _of_ future's References _to_ their potential final values. A future is empty (Nothing) until it is resolved (filled with 'Just value'), hence its type 'Maybe Ref'. A future reference must be final, and this must be guaranteed by the runtime system.

type Cont = () -> Stmt

We have a single (universal) type for our continuations. Later, if we introduce local-variables we are going to need an extra type for Continuations: 'Ref -> Stmt'

newtype Proc

Each process is a triple of the this object, its destiny, and its (resumable) continuation (note: is a newtype just for overriding its Show instance, check module PP)

type ProcTable = Map ObjRef [Proc]

The (global) scheduler's runtime Process Table. It is a mapping _from_ an object-reference _to_ its queue of processes

data BExp

A boolean expression occurs only as a control-flow predicate (if & while) It does only reference equality of attributes (BEq) and combinators on them (conjuction,disjunction,negation).
